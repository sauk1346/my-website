# Asesoría 1: Introducción

## 1. Herramientas Python

Hay varias herramientas sugeridas para que nuestro código siga los estándares `PEP8`. Entre ellas tenemos

- `Black`: Formateador automático de código Python
- `Autopep8`: Formateador automático de código Python
- `Isort`: Ordena y organiza las importaciones de librerías

Analizadores estáticos de código:

- `Flake8`
- `Pylint`
- `Mypy`: analizador de tipos

## 2. Git

**Configuración inicial**

- Lo primero es establecer el nombre de usuario y dirección de correo electrónico.
- Esto es importante poruqe los "commits" de Git usan esta información y es introducida de manera inmutable en los commits que envías.

```sh
git config --global user.name "John Doe"
git config --global user.mail johndoe@example.com 
```

- Para verificar tu configuración, se utiliza el siguiente comando:

```sh
git config --list
```

## 3. pre-commit
Es una herramienta que ejecuta scripts automáticamente antes de que se realice un commit. Estos scripts se llaman `hooks` y pueden ayudarte a:

- Formatear tu código automáticamente (por ejemplo, con black o isort)
- Verificar si tu código sigue el estándar PEP8 (flake8, pylint)
- Evitar que subas archivos grander por error
- Bloquear secretos como claves API en tus commits
- Validar que los archivos .json, .yms, etc., estén bien formados

```sh
pip install pre-commit
pre-commit install
```
