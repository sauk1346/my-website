# Infraestructura Global de AWS

La nube de AWS abarca 120 Zonas de disponibilidad dentro de 38 regiones geográficas, con planes anunciados para 10 más zonas de disponibilidad y 3 más regiones de AWS en Arabia Saudita, Chile y la Nube soberana europea de AWS

## 1. Regiones de AWS: Alta disponibilidad y conectividad
- Una región de AWS es una zona geográfica que contiene múltiples centros de datos
- Usted controla la replicación de datos entre regiones
- La comunicación entre regiones viaja por la red troncal global de AWS
- Cada región ofrece redundancia y conectividad de nivel empresarial
- Una región normalmente consta de 2 o más Zonas de Disponibilidad (AZs)

## 2. Cómo elegir la región adecuada en AWS

- Gobernanza de datos y requisitos legales
- Servicios disponibles dentro de la región
- Proximidad con los clientes (latencia)
- Costos (varían según la región)

## 6. Servicios de Cómputo en AWS: Flexibilidad y Escalabilidad

Los servicios de cómputo en AWS permiten ejecutar aplicaciones de manera flexible, desde instancias virtuales tradicionales hasta entornos serverless y contenedores.

Con ellos, puedes escalar automáticamente, optimizar costos y elegir el modelo de despliegue que mejor se adapte a tus cargas de trabajo.

- Amazon C2
- Amazon EC2 Auto Scaling
- Amazon Elastic Container Service (Amazon ECS)
- Amazon EC2 Container Registry
- AWS Elastic Beanstalk
- AWS Lambda
- Amazon Elastic Kubernetes Service (Amazon EKS)
- AWS Fargate
