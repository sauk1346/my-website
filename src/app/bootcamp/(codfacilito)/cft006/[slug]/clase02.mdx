# Guardar y administrar cambios

## 1. Pasos para creación proyecto

1. Crear repositorio remoto
    ```text
    https://github.com/new
    ```
2. Crear repositorio local
    ```sh
    git init
    ```
3. Vincular ambos repositorio
    ```sh
    git remote add origin <link_repositorio_remoto>
    ```


## 2. Stash
Es un comando poderoso que le permite guardar temporalmente (o "guardar") los cambios que ha realizado en su directorio de trabajo para que pueda trabajar en otra cosa y luego regresar y volver a aplicarlos más tarde.

## 3. Flujo de trabajo

### Preparar cambios para commit
- Agregar todos los archivos modificados al al "área de preparación" (staging)
    ```sh
    git add .
    ```
- Agregar un archivo específico
    ```sh
    git add <nombre_archivo>
    ```

### Deshacer cambios del área de preparación
- Quitar todos los archivos del staging (sin perder cambios)
    ```sh
    git reset
    ```
- Quitar un archivo específico del staging (sin perder cambios)
    ```sh
    git reset <nombre_archivo>
    ```

### Guardar cambios en el repositorio
- Crear un commit con los archivos del staging
    ```sh
    git commit -m "Descripción de los cambios"
    ```

### Ver el estado actual
- Verificar qué archivos están modificados o en staging
    ```sh
    git status
    ```
