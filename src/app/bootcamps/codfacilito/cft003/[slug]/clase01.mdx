# Introducción a Git

Git es un sistema de control de versiones.

## Funcionamiento de Git

- **Área de trabajo (Working Directory)**: Es la carpeta del proyecto donde trabajas directamente con los archivos. Aquí realizas modificaciones, creas nuevos archivos o eliminas existentes.
- **Área de preparación (Staging Area)**: Conocida como *staging*. Es un área intermedia donde se almacenan los cambios que quieres incluir en el próximo commit. Los archivos aquí están listos para ser confirmados pero aún no han sido guardados permanentemente en el historial.
- **Repositorio (Repository)**: Es donde Git almacena permanentemente el historial completo de cambios del proyecto. Contiene todos los commits, ramas, etiquetas y metadatos. Puede ser local (en tu computadora) o remoto (como en GitHub, GitLab, etc.).

**Flujo típico:**

1. Modificas archivos en el área de trabajo
2. Añades los cambios al área de preparación con `git add`
3. Confirmas los cambios al repositorio con `git commit`

## ¿Qué son los commits?

- Es la unidad fundamental en Git, la cual representa un registro de cambios realizados en el proyecto.
- Podemos verlo como un snapshot del proyecto.

## Commando básicos

|Comando|Descripción|
|---|---|
|`git init`|Permite inicializar una carpeta en un repositorio|
|`git clone`|Clonar (descargar) un proyecto remoto a local|
|`git status`|Ver si hay cambios en el proyecto|
|`git add .`|Guardar todos los cambios del directorio actual|
|`git branch`|Para saber en qué rama estamos|
|`git merge`|Combina los cambios de una rama con otra rama|


## Ejemplo Proyecto

1. Creación área de trabajo
```sh
mkdir mi-proyecto
```

2. Creación repositorio

```sh
cd mi-proyecto
git init
```

3. Crear un archivo (README.md)

```sh
echo "mi primer proyecto" > README.md
```

4. Guardar cambios al área de staging

```sh
git add .
git commit -m "add readme file"
```
