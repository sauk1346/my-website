# Tuplas
Son objetos **inmutables**, no se pueden modificar una vez creadas

```python
settings = ("localhost", 3306, True)

print(settings)
print(type(settings))
```
> ('localhost', 3306, True)
> \<class 'tuple'\>

## 1. Crear tuplas

Es posible omitir los parentesis al crear una tupla

```python
tupla_1 = 1, 2, 3, 4, 5
tupla_2 = 10,

print(tupla_1)
print(type(tupla_1))
print(tupla_2)
print(type(tupla_2))
```
> (1, 2, 3, 4, 5)
> \<class 'tuple'\>
> (10,)
> \<class 'tuple'\>

## 2. Desempaquetado de Tuplas

```python
courses = ("Python", "Django", "Ruby", "Ruby on Rails", "MySQL")

var1, var2, var3, var4, var5 = courses

print(var1)
print(var2)
print(var3)
print(var4)
print(var5)
```
> Python
> Django
> Ruby
> Ruby on Rails
> MySQL

## 3. Desempaquetado incompleto de Tuplas

- A veces no se desea guardar todos los valores de la tupla 
- Para esto, se utiliza la convención  guión bajo `_`


```python
courses = ("Python", "Django", "Ruby", "Ruby on Rails", "MySQL")

var1, _, var3, var4, _ = courses

print(var1)
print(var3)
print(var4)
```
> Python
> Ruby
> Ruby on Rails

## 4. Agrupar Valores
Para esto se utiliza el caracter arterístco `*`

```python
courses = ("Python", "Django", "Ruby", "Ruby on Rails", "MySQL")

var1, var2, *sub_courses, last_value = courses

print(var1)
print(var2)
print(sub_courses)
print(last_value)
```
> Python
> Django
> \['Ruby', 'Ruby on Rails'\]
> MySQL

## 5. Función zip

Permite trabajar con diferentes colecciones para generar tuplas.

```python
users = ["user1", "user2", "user3"]
courses = ("Python", "Django", "Rails")

response = list(zip(users,courses))

print(response)
```
> \[('user1', 'Python'), ('user2', 'Django'), ('user3', 'Rails')\]

## 6. Funciones de Tuplas

|Función|Descripción|
|---|---|
|`len(i)`|Devuelve el **número de elementos** de la **colección-i**|
|`sorted(i)`|Devuelve una **lista ordenada ascendente** de la **colección-i** |
|`count(i)`|Devuelve el **número de veces** que aparece el **elemento-i** en la **colección base**|
