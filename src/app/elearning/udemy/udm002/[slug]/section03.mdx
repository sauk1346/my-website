# Sección 03: Big O

## ¿Qué es un Buen Código?
- Un código que complete la tarea asignada, pero que también sea **eficiente**.
- Esto significa que importa cuánto tiempo se demora un algoritmo en resolver un problema.


**Ejemplo**

```js
const everyone = ['dory', 'bruce', 'marlin', 'nemo', 'gill', 'bloat', 'nigel', 'squirt', 'darla', 'hank'];

function findNemo(array) {
    let t0 = performance.now();
    for (let i = 0; i < array.length; i++) {
        if (array[i] === 'nemo') {
            console.log('NEMO ENCONTRADO!');
        }
    }
    let t1 = performance.now();
    console.log('Encontrar a Nemo tomó: ' + (t1-t0) + ' milisegundos')
}

findNemo(everyone);
```

> Encontrar a Nemo tomó: 5.1937229999999985 milisegundos

## Big O

### O(n)
En el ejemplo de "encontrar a nemo" su tiempo de ejecución es `O(n)`. Esto se debe a que:
- Si el arreglo fuese de 2 elementos, tardaría 2 operaciones en el peor caso. (último elemento)
- Si el arreglo fuese de 3 elementos, tardaría 3 operaciones en el peor caso. (último elemento)
- ...
- Si el arreglo fuese de `n` elementos, tardaría `n` operaciones en el peor caso. (último elemento)

### O(1)
Es un tiempo de ejecución constante, que no depende de la cantidad de operaciones

**Ejemplo 1**

```js
// O(1)
console.log("hola mundo!");
```

## Ejercicio 01
Calcular Big O

```js
function funChallenge(input) {
    let a = 10;
    a = 50 + 3;

    for (let i = 0; i < input.length; i++) {
        anotherFunction();
        let stranger = true;
        a++;
    }
    return a;
}
```

<details>
    <summary>Solución</summary>
```js
function funChallenge(input) {
    let a = 10; // O(1)
    a = 50 + 3; // O(1)

    for (let i = 0; i < input.length; i++) {
        anotherFunction(); // O(n)
        let stranger = true; // O(n)
        a++; // O(n)
    }
    return a; // O(1)
}

// O(1+1+1+n+n+n+n) = O(3+3n) = O(n) 
```
</details>
