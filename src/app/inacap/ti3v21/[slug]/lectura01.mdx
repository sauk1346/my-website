# Conceptos Básicos de POO

## POO
- OOP: **Oriented Object Programming**
- Es un paradigma de la programación, radicalmente diferente al enfoque clásico

## Clases
- Es similar a una plantilla. A partir de la clase, se crearán objetos
- Ejemplo: Clase Auto, puede crear distintos objetos autos, todos diferentes unos de los otros

### Jerarquía de Clases
- Las clases de más arriba se llaman **super-clases** o **clases-padre**
- Las clases de más abajo se llaman **sub-clases** o **clases-hijos**
- Las clases padres engloban una idea más general
- Las clases hijos son más detalladas

<img 
    src="/ti3v21/images/fig09.png"
    style={{ display: 'block', margin: '0 auto', width:'90%' }} 
/>

## Objetos
- Si las Clases son las "plantillas", los Objetos son la creación de esas "plantillas"
- Otro nombre que recibe un objeto es **instancia** de una clase
- Un objeto tiene **atributos** y **métodos**
	- atributos: son características (análogo a variables)
	- métodos: son acciones (análogo a funciones)

<img 
    src="/ti3v21/images/fig10.png"
    style={{ display: 'block', margin: '0 auto', width:'90%' }} 
/>

## Herencia
- Es una característica de las Clases
- Una clase-padre puede heredar sus **atributos** y **métodos** a sus clases-hijos
- Esto permite que la clase-hijo no comience desde cero

<img 
    src="/ti3v21/images/fig11.png"
    style={{ display: 'block', margin: '0 auto', width:'90%' }} 
/>

## Polimorfismo
- Se refiere a sobreescribir los métodos de un objeto
- Esto se aplica, ya que para un objeto1, un mismo método de utiliza de forma distinta que para un objeto2

<img 
    src="/ti3v21/images/fig12.png"
    style={{ display: 'block', margin: '0 auto', width:'90%' }} 
/>
<br/>
- En el ejemplo, los métodos `avanzar()` y `detenerse()` responden de forma distinta para cada objeto

## Clases Abstractas
- Se pueden definir métodos y atributos, pero no se pueden instanciar objetos
- Son usadas para construir subclases y con éstas instancias objetos


# Actividad de Práctica 01

1. **¿Qué es una clase en programación orientada a objetos?**
	1. <span style={{color:'red'}}>Un modelo o plantilla para crear objetos.</span>
	2. Una instancia de un objeto.
	3. Un conjunto de variables.
	4. Un método que realiza operaciones.

2. **¿Cuál es el propósito de la herencia en programación orientada a objetos?**
	1. Definir variables en una clase.
	2. Crear múltiples instancias de una clase.
	3. <span style={{color:'red'}}>Permitir que una clase herede atributos y métodos de otra.</span>
	4. Combinar varias clases en una.

3. **¿Qué es el polimorfismo en programación orientada a objetos?**
	1. La capacidad de una clase para heredar de múltiples clases.
	2. La capacidad de una clase para tener múltiples instancias.
	3. La capacidad de una clase para cambiar su estructura interna.
	4. <span style={{color:'red'}}>La capacidad de una calses para tomar diferentes formas y comportarse de manera diferente.</span>

4. **¿Qué son los atributos en programación orientada a objetos?**
	1. Funciones que realizan operaciones.
	2. <span style={{color:'red'}}>Variables que almacenan datos.</span>
	3. Métodos que definen la estructura de una clase.
	4. Instrucciones para crear objetos.

5. **¿Cuál es el propósito de los métodos en programación orientada a objetos?**
	1. Definir variables en una clase.
	2. Almacenar datos en una instancia.
	3. <span style={{color:'red'}}>Realizar operaciones o acciones en un objeto.</span>
	4. Crear instancias de una clase.
