# Actividad de Práctica 05

1. **¿Cuál es el propósito principal de un Diagrama de Clases en UML?**
	1. <span style={{color:'red'}}>Visualizar la estructura estática y relaciones entre clases</span>
	2. Representar el flujo de control en un programa
	3. Mostrar la ejecución secuencial de métodos
	4. Indicar el tiempo de ejecución de cada operación

2. **¿Qué elemento de un Diagrama de Clases representa una propiedad o característica de una clase?**
	1. Operación
	2. Método
	3. <span style={{color:'red'}}>Atributo</span>
	4. Relación de Agregación

3. **¿Qué tipo de relación indica que una clase está compuesta por otras clases?**
	1. La Herencia
	2. La Asociación
	3. La Dependencia
	4. <span style={{color:'red'}}>La Composición</span>

4. **¿Cuál es el propósito de la herencia en un Diagrama de Clases?**
	1. Mostrar una relación de dependencia entre clases
	2. <span style={{color:'red'}}>Permitir a una clase compartir propiedades y comportamientos de otra</span>
	3. Indicar una relación de todo o parte entre clases
	4. Representar una relación temporal entre clases

5. **¿Cuál es el símbolo utilizado para representar una relación de asociación en un Diagrama de Clases?**
	1. Flecha con rombos
	2. Flecha sólida
	3. <span style={{color:'red'}}>Línea continua</span>
	4. Línea punteada
