# Diseño de diagramas de clases y casos de uso para la solución

## Diagramar antes de codear
1. Visualización de la estructura y el comportamiento del sistema
2. Comunicación efectiva entre equipos y partes interesadas
3. Identificación de requisitos y decisiones de diseño
4. Facilitación del mantenimiento y la escalabilidad

# Actividad de práctica 15

1. **¿Cuál de los siguientes enunciados es el propósito principal de diseñar diagramas de clases y casos de uso utilizando la notación UML?**
	1. <span style={{color:'red'}}>Representar la estructura y el comportamiento del sistema.</span>
	2. Establecer los requisitos de hardware para el sistema.
	3. Documentar el proceso de prueba del software.
	4. Definir los colores y estilos de la interfaz de usuario.

2. **De las siguientes opciones, ¿Cuál representa un diagrama de clases en UML?**
	1. El flujo de interacción entre los usuarios y el sistema.
	2. Los casos de prueba para el sistema.
	3. <span style={{color:'red'}}>La estructura estática del sistema, incluyendo clases, atributos y relaciones.</span>
	4. Las operaciones de mantenimiento del sistema.

3. **¿Cuál de los siguientes enunciados es el propósito principal de un diagrama de casos de uso en UML?**
	1. Definir la implementación técnica del sistema.
	2. Describir la lógica de negocio del sistema.
	3. Detallar la estructura de datos del sistema.
	4. <span style={{color:'red'}}>Identificar los actores y sus interacciones con el sistema.</span>

4. **¿Qué enunciado de los siguientes representa un actor en un diagrama de casos de uso en UML?**
	1. Una clase del sistema.
	2. <span style={{color:'red'}}>Un usuario o sistema externo que interactúa con el sistema.</span>
	3. Una relación de herencia entre clases.
	4. Un atributo de una clase.

5. **¿Cuál de las siguientes afirmaciones es uno de los principales beneficios de diseñar diagramas de clases y casos de uso en UML?**
	1. Determina los requisitos de hardware del sistema.
	2. Minimiza los errores de codificación.
	3. <span style={{color:'red'}}>Facilita la comunicación entre los miembros del equipo.</span>
	4. Define los colores y estilos de la interfaz de usuario.
