- Se utiliza para describir una gran historia de usuario que será dividio en historias más pequeñas

## User Stories
Representación de un requisito del usuario en forma escrita, en una o dos frases, utilizando el lenguaje común del usuario

### INVEST
Una User Story debe cumplir con las características del modelo INVEST, las cuales garantizan que éstas sean claras, manejables y valiosas

||Significado|Descripción|
|---|---|---|
|**I**|Independencia (Independent)|La *User Story* debe ser autónoma y no depender de otras historias, para así evitar bloqueos en el desarrollo|
|**N**|Negociable (Negotiable)|La *User Story* no debe ser un contrato rígido, sino un pto. de partida para la conversación entre el equipo y el cliente o Product Owner|
|**V**|Valiosa (Valuable)|La *User Story* debe aportar valor al cliente o al negocio|
|**E**|Estimable (Estimable)|*User Story* debe ser lo suficientemente clara y concreta para permitir que el equipo estime el esfuerzo necesario|
|**S**|Pequeña (Small)|La *User Story* debe ser lo suficientemente pequeña para completarse en un sprint o iteración|
|**T**|Verificable (Testable)|La *User Story* debe poder verificarse mediante criterios de aceptación claros|

## Task
- Representación del requisito que está en lenguaje del usuario, pero de una forma técnica
- Está definido cómo se va a trabajar y quiénes van a participar

### SMART
Proporciona un marco para definir tareas (tasks) claras, alcanzables y efectivas dentro de un proyecto. Este modelo ayuda a garantizar que las tareas estén bien definidas y alineadas con los objetivos generales

||Significado|Descripción|
|---|---|---|
|**S**|Específica (Specific)|La tarea debe estar claramente definida, sin ambigüedades|
|**M**|Medible (Measurable)|La tarea debe incluir criterios que permitan evaluar el progrso o el éxito|
|**A**|Alcanzable (Achievable)|La tarea debe ser realista, teniendo en cuenta los recursos y las limitaciones disponibles|
|**R**|Relevante (Relevant)|La tarea debe estar alineada con los objetivos generales del proyecto o negocio|
|**T**|Temporal (Time-bound)|La tarea debe tener un plazo claro para su finalización|
