# Mockups

## ¿Qué es?
- Representaciones visuales estéticas
- Permite esbozar y diseñar la apariencia y la interacción de una aplicación o sistema de software antes que se inicie la programación
- Ofrece un enfoque pragmático y altamente efectivo para capturar los requisitos visuales y funcionales
- Permite a los equipos de desarrollo, diseñadores y stakeholders colaborar de manera más eficiente desde las etapas iniciales del proyecto.

## Diseño de Mockups
1. Comprender los requisitos
2. Identificar las páginas claves
3. Seleccionar herramientas de diseño
4. Definir la Estructura y Navegación

## Conceptos Fundamentales de UX/UI

### Experiencia de Usuario (UX)

- Se refiere a cómo se siente y percibe un usuario al interactuar con un sistema, producto o servicio
- El objetivo principal de UX es proporcionar una experiencia positiva y satisfactoria para los usuarios

|||
|---|---|
|Investigación del usuario|Comprender las necesidades y comportamientos de los usuarios a través de investigación, encuestas y análisis|
|Diseño centrado en el usuario|Colocar en el centro del proceso de diseño, asegurándose de que el producto resuelva problemas reales y sea fácil de usar|
|Arquitectura de la información|Organizar la estructura y navegación del producto para que sea intuitiva y fácil de entender|
|Prototipos y pruebas|Crear prototipos para evaluar conceptos y realizar pruebas con usuarios para obtener comentarios valiosos|
|Usabilidad|Garantizar que el producto sea fácil de usar y que los usuarios puedan completar tareas sin dificultad|
|Accesibilidad|Asegurarse de que el producto sea accesible para usuarios con diversas habilidades y discapacidades|
|Diseño emocional|Considerar cómo el diseño afecta las emociones y experiencias de los usuarios|
|Retroalimentación continua|Recopilar y utilizar constantemente comentarios de los usuarios para mejorar la experiencia con el tiempo|

### Interfaz de Usuario (UI)
- Se centra en el diseño visual y la presentación del producto
- Mientras UX aborda la experiencia general, UI se centra específicamente en cómo se ve y se presenta el producto

|||
|---|---|
|Diseño Visual|Crear una apariencie atractiva y coherente que represente la identidad de la marca|
|Diseño de Interacción|Diseñar cómo los usuarios interactúan con los elementos visuales, como botones, menús y formularios|
|Tipografía y Color|Seleccionar tipografías y colores que mejoren la legibilidad y la estética|
|Iconografía|Utilizar iconos de manera efectiva para representar funciones y acciones de manera visual|
|Diseño Responsivo|Garantizar que el producto sea fácil de usar y que los usuarios puedan completar tareas sin dificultad|
|Animaciones y Transiciones|Para mejorar la experiencia visual y la retroalimentación del usuario|
|Consistencia|Mantener un diseño coherente en toda la interfaz para que los usuarios puedan predecir y entender mejor la navegación|
|Prototipos y Visuales|Crear prototipos de diseño para visualizar cómo se verá el producto final|
