### ¿Cómo se intepreta una matriz de riesgo?

1. **Riesgo alto:** Requiere acciones inmediatas, como parches, actualizaciones o medidas de mitigación urgentes
2. **Riesgo medio:** Necesita un plan de acción a corto plazo, incluyendo posibles cambios en la configuración o mejoras en la seguridad
3. **Riesgo bajo:** Puede ser monitorizado y abordado en el ciclo de mantenimiento regular

### ¿Cuáles son las acciones posteriores?

1. **Mitigación:** Se deben implementar soluciones para reducir el riesgo (parches, firewalls, etc.)
2. **Monitoreo:** Requiere establecer sistemas de monitoreo continuo para detectar y responder a incidentes
3. **Revisión regular:** Implica actualizar la matriz de riesgo periódicamente para reflejar nuevos activos, vulnerabilidades y cambios en el entorno

## Identificación y evaluación de vulnerabilidades mediante CVSS

*Common Vulnerability Scoring System* (CVSS) es una herramienta estándar para evaluar la gravedad de las vulnerabilidades de seguridad. Utilizarla hace posible desarrollar políticas de prevención de riesgos efectivas para minimizar el impacto de estas vulnerabilidades.

### Fundamentos de CVSS

1. **Métricas Base:** Representan las características intrínsecas de una vulnerabilidad que no cambian con el tiempo ni con el entorno de implementación
    - **Vectores de Ataque:** cómo una vulnerabilidad puede ser explotada
    - **Complejidad del Ataque:** dificultad de explotar la vulnerabilidad
    - **Privilegios Requeridos:** nivel de acceso necesario para explotar la vulnerabilidad
    - **Interacción del Usuario:** indica si es necesaria la interacción del usuario para que la vulnerabilidad sea explotada
    - **Alcance:** describe si la vulnerabilidad puede afectar componentes más alla de su propio entorno de seguridad
    - **Impacto:** evalua si el impacto potencial en la confidencialidad, integridad y disponibilidad
2. **Métricas Temporales:** Reflejan las características de una vulnerabilidad que pueden cambiar con el tiempo, tales como la disponibilidad de parches o soluciones alternativas
    - **Exploit Code Maturity:** evalúa la disponibilidad de código de explotación (disponible, teórico, funcional)
    - **Remediation Level:** indica el nivel de correción disponible para la vulnerabilidad (ninguna, solución temporal, parche oficial)
    - **Report Confidence:** mide la confianza en la existencia y detalles técnicos de la vulnerabilidad (desconocido, razonable, confirmado)
3. **Métricas Ambientales:** permiten personalizar la puntuación CVSS según las características específicas del entorno de la organización
    - **Confidentiality Requirement:** la importancia de la confidencialidad en el entorno específico (bajo, medio, alto)
    - **Integrity Requirement:** la importancia de la integridad en el entorno específico (bajo, medio, alto)
    - **Availability Requirement:** la importancia de la disponibilidad en el entorno específico (bajo, medio, alto)
    - **Modified Base Metrics:** permite ajustar las métricas en función de las condiciones específicas del entorno
4. **Vectores y fórmulas:** CVSS utiliza un vercto de cadena que representa una combinación de todas las métricas mencionadas anteriormente. Este vector se utiliza para calcular una puntuación numérica que indica la severidad de la vulnerabilidad. La puntuación CVSS se expresa en una escala 0 (baja severidad) a 10 (máxima severidad) 
5. **Consistencia y comparabilidad:** CVSS proporciona una metodología estándar que asegura consistencia en la evaluación de vulnerabilidades, lo cual facilita la comparación entre diferentes vulnerabilidades y contextos
