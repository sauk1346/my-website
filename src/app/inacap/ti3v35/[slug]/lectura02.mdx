# Interconexión de dispositivos y redes

## Protocolo TCP/IP (stack)
Es el protocolo usado en internet, por lo que debe estar incluido en cualquier computador que quiera utilizar cualquier servicio de internet.

En la práctica, TCP/IP **no es un único protocolo**, sino que es un stack (un conjunto de protocolos).

### Capas
El protocolo TCP/IP, se divide en 4 capas. Cada capa soluciona un problema en la comunicación y tiene asociado uno o varios protocolos para ello.

1. **Aplicación:** Incluidos los protocolos destinados a proporcionar servicios
    - **FTP** (transferencia de ficheros)
    - **HTTP** (navegación en internet)
    - **SMTP** (correo electrónico)
    - **SSH** (conexión segura)
2. **Transporte:** Proporciona el transporte de los datos con fibilidad suficiente
    - **TCP** (*transfer control protocol*)
    - **UDP** (*user datagram protocol*)
    - **TLS** (*transport layer security*)
3. **Red:** Se encarga de enviar cada paquete a su destino (enrutar datos)
    - **IP** (*internet protocol*)
    - **ICMP** (*internet control message protocol*)
    - **IGMP** (*internet group management protocol*)
    - **ARP** (*address resolution protocol*)
    - **RARP** (*reverse address resolution protocol*)
4. **Enlace:** Se encarga de la transmisión a través del medio físico. También presenta algoritmos de control y correción de errores.


## Protocolo TCP
Es un protocolo de control de transmisión de la **capa de transporte**.

**Características**

- Orientado a la conexión
- Fiable
- Control de errores

## Puertos
- Un puerto se utiliza para distinguir entre los distintos destinos, dentro de un mismo host al que va dirigida la información.
- Cada aplicación utiliza un número de puerto distinto. Cuando una aplicación está esperando un mensaje, lo hace en un puerto determinado y se dice que está en "estado listen" o "a la escucha".
- Un puerto es un número de **16 bits**, por lo que existen $2^{16} = 65536$ números de puertos posibles en cada host. Las aplicaciones utilizan estos puertos para enviar y recibir mensajes.  
- Los **primeros 256 puertos** son llamados **"puertos bien conocidos" (well known)**, y se usan para servicios comunes, como HTTP, FTP, etc.
- **TCP** asigna los números de puerto "bien conocidos" para aplicaciones servidoras (que ofrecen servicios). 
- El resto de los números disponibles se asignan a las aplicaciones cliente (que solicitan servicios)

|Puerto|Aplicación|
|---|----|
|0 - 255|Aplicaciones públicas|
|255 - 1023|Aplicaciones comerciales|
|1023 - adelante|No están reguladas|

Los siguientes puertos son controlados y asignados por IANA (*Internet Assigned Numbers Authority*)

| Protocolo | Puerto | Tipo | Descripción |
|-----------|--------|------|-------------|
| echo | 7 | TCP/UDP | Eco (devuelve lo que recibe) |
| daytime | 13 | TCP/UDP | Fecha y hora |
| ftp | 21 | TCP | Protocolo de Transferencia de Ficheros |
| telnet | 23 | TCP | Terminal remoto |
| smtp | 25 | TCP | Protocolo de correo simple |
| domain (DNS) | 53 | TCP/UDP | Servidor de nombres de dominio |
| www-http | 80 | TCP | World Wide Web HTTP (páginas web) |
| pop3 | 110 | TCP | Post Office Protocol - Versión 3 (recibir correo) |

## Protocolo UDP
*User Datagram Protocol*. Proporciona una comunicación sencilla entre dos hosts y que no consume muchos recursos. 

**Características**

- No confiable
- No orientado a conexión
- No cuenta con control de errores


Es un protocolo útil cuando no es necesario mucho control de los datos enviados. Se utiliza cuando **la rapidez es más importante que la calidad**.

Por ejemplo, es útil para los juegos online eliminando los LAG de retardo.

## Direccionamiento IP
Para poder identificar una máquina en internet, cada una debe tener una dirección IP (*intener protocol*) asignada, la cual es asignada por IANA, encargada de administrar las direcciones IP públicas.

Las IP son un **dígito binario de 32 bits**, representadas en decimales de 4 octetos (A.B.C.D)

|Notación|A|B|C|D|
|---|---|---|---|---|
|Binaria|00000001|11111110|11000000|11110000|
|Decimal|1|254|192|240|


Todas las direcciones IP de un host tienen 2 partes:

1. Identifica la red a la que pertenece el host
2. El ordenador dentro de la red en la que se encuentra
