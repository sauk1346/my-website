### Fundamentos del patrón MVC
El **patrón de arquitectura MVC** (modelo-vista-controlador) es un modelo de organización del código que busca **separar responsabilidades** dentro de una aplicación.

Esta separación permite:

- Mantener el código más limpio
- Reutilizar componentes
- Facilitar el mantenimiento y trabajo en equipo

<SmartTable
	tableLayout="auto"
	headers={['Componente', 'Descripción']}

		['**Vista**', 'Define lo que el usuario ve (HTML, CSS, imágenes, formularios)'],
		['**Controlador**', 'Actúa como puente entre el modelo y la vista. Recibe las solicitudes del usuario, obtiene los datos del modelo y los envía a la vista']

/>

**Ejemplo**

- **Modelo:** Libro (título, autor, stock)
- **Controlador:** Busca todos los libros disponibles y decide mostrarlos
- **Vista:** Muestra una tabla con los títulos de los libros en formato HTML

## Django MVC en Python

Django es un framework de desarrollo web de código abierto construido en Python
Creado para acelerar la construcción de aplicaciones robustas, seguras y altamente escalables. Gracias a su arquitectura de “baterías incluidas”, ofrece un potente *Object-Relational Mapping* (ORM), sistema de plantillas, enrutamiento de URL, gestión de formularios, autenticación de usuarios y panel de administración automático, lo que reduce drásticamente el tiempo de configuración y refuerza la seguridad frente a ataques comunes *Cross-Site Request Forgery* (CSRF) , inyección *Structured Query Language* (SQL) y *Cross-Site Scripting* (XSS).

Empresas como *Instagram*, *Pinterest* y *Mozilla* confían en Django para proyectos de gran escala, mientras que su curva de aprendizaje accesible lo convierte en una opción ideal para desarrolladores, startups y entornos académicos. Su enfoque en el principio **DRY** (*Don’t Repeat Yourself*) y la separación clara de responsabilidades permiten codebases limpias, fáciles de mantener y escalables a largo plazo.

### ¿Qué hace exactamente un framework?
Un framework es un conjunto de herramientas, estructuras y componentes predefinidos que te ayudan a desarrollar software de forma más eficiente. En lugar de programar todo desde cero, puedes apoyarte en funcionalidades ya construidas, seguras y probadas.

En el caso de Django, te ahorra trabajo en tareas como:

- Enrutamiento de URL
- Conexión y gestión de bases de datos
- Validación de formularios
- Control de sesiones y usuarios
- Generación de contenido HTML dinámico

> Django sigue el patrón MTV (modelo-template-vista), que es **prácticamente equivalente** al MVC

## Estructura General de un Proyecto Django

### 1. Proyecto Django
Es la estructura general del sitio web o sistema. Cuando ejecutas:

```sh
django-admin startproject mi_sitio
```

se crea una carpeta con archivos de configuración global, como:

	tableLayout="auto"

		['`urls.py`', 'Contiene las rutas principales del sitio'],
		['`wsgi.py` y `asgi.py`', 'Permiten que Django se comunique con servidores web']
	]}
/>

### 2. Aplicaciones (apps)

- Un proyecto Django puede **agrupar múltiples** "apps", cada una encargada de una funcionalidad concreta y modular (ej: blog, carrito de compras o sistema de comentarios)
- Para crear una nueva app llamada **tienda**, ejecuta:

```sh
python manage.py startapp tienda
```

### 3. Modelos (models.py)
Aquí tienes las **tablas de tu base de datos** mediante **clases de Python**. Django se encarga de traducirlas automáticamente a SQL.

### 4. Vistas (views.py)
Las vistas definen la **lógica de la aplicación**. Es decir, qué se hace cuando el usuario visita una URL.

### 5. Plantillas (templates/)
Archivos HTML enriquecidos con etiquetas dinámicas que **insertan datos del servidor en el frontend**. Su integración con vistas y modelos permite:

- Definir estructuras de presentación reutilizables
- Renderizar contextos enviados desde la vista
- Mantener una separación clara entre lógica y presentación
