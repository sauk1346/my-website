# Requerimientos del Software

**Material Clase:** [clase05.pdf](/inacap/ti3v43/docs/clase05.pdf)


## 1. ¿Qué son los Requerimientos de Software?
Son especificaciones detalladas que describen qué debe hacer un sistema de software, y bajo qué restricciones debe operar.

### Requerimientos Funcionales
- Describen las funciones específicas que el sistema debe realiza.
- **Ejemplo 1:** "El sistema debe permitir a los usuarios registrarse con email y contraseña"
- **Ejemplo 2:** "El sistema debe generar reportes mensuales de ventas"

### Requerimientos No Funcionales
- Especifican criterios de calidad y restricciones operacionales como rendimiento, usabilidad, confiabilidad y escalabilidad.
- **Ejemplo 1:** "El sistema debe responder en menos de 2 segundos"
- **Ejemplo 2:** "Debe soportar 100 usuarios concurrentes"

>Los **negocios** expresan la necesidad desde la perspectiva del cliente


## 2. Técnicas de Levantamiento de Requerimientos

- Entrevistas con usuarios y expertos
- Cuestionarios y encuestas
- Observación directa
- Talleres de co-creación
- Historias de usuario

## 3. UML
- Lenguaje Unificado de Modelado
- Estándar para representar requerimientos y diseños
- Permite comunicación entre cliente y desarrolladores
- Diagramas más usados:
    - Casos de uso
    - Clase
    - Secuencia
    - Actividad

### Ejemplo

- Sistema de "Reserva de Canchas"
- Opciones:
    - Reservar cancha
    - Pagar en línea
    - Cancelar reserva
