## 2. Ciclo de vida del producto de software
Los ciclos de vida de un "producto" y "proyecto" se encuentra estrechamente relacionados

- Cualquier idea vinculada con un producto de software tiene que ser conducida por un proyecto asociado

### 3. Metodologías de desarrollo de software

### Método cascada

- Investigación y especificaciones
- Diseño
- desarrollo
- Testeo final

Inicialmente se hace una toma de requisitos del proyecto completa y se establece una **secuencia de trabajo**, esto se lleva a cabo **una única vez** y al principio del proyecto

Las metodologías tradicionales son ideales para proyectos sobre los cuales se tiene mucha experiencia os e **desarrollan bajo unas condiciones predecibles y estables**.

## 3. Metodologías ágiles
- Cada una de las fases, se dividen en fases aún más pequeñas llamadas sprints.
- Cada uno de estos sprints se puede completar de manera independiente a la fase que se encuentra, y abrirá las puertas a comenzar con otras fases, aunque la suya propia no esté terminada de forma completa.

Las metodologías ágiles o scrum, se caracterizan por ser **ser adaptativas y flexibles**, lo cual significa que **permite evolucionar el proyecto de una manera más ágil en función de nuestras necesidades**.
