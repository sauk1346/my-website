# Modelado de sistemas mediante diagramas UML

## 1. Creación de diagramas para un prototipo

El modelado de sistemas mediante diagramas UML (*Unified Modeling Language*) es una práctica esencial en el **diseño y desarrollo de sistemas de software**. 

Este enfoque permite representar gráficamente la estructura, el comportamiento y las interacciones del sistema, facilitando la comprensión, comunicación y colaboración entre los miembros de un equipo.

## 3. Pasos para crear Diagramas UML

<SmartTable
    tableLayout="auto"
    headers={["Pasos", "Descripción"]}
    rows={[
        [
            "**Identificación de requisitos**",
            "Recolección y análisis de requisitos funcionales y no funcionales, priorización de los más importantes"
        ],
        [
            "**Selección de diagramas apropiados**",
            "Elegir los diagramas más adecuados según las características del sistema y el objetivo del prototipo"
        ],
        [
            "**Asignación de roles y funciones**",
            "Distribución de tareas y adopción de metodologías ágiles o tradicionales"
        ],
        [
            "**Construcción de diagramas**",
            "Definición de elementos y relaciones, uso de herramientas de modelado como Lucidchart, StarUML o Visual Paradigm"
        ],
        [
            "**Validación de diagramas**",
            "Revisión interna, retroalimentación del equipo, prueba del prototipo"
        ],
        [
            "**Documentación y entrega**",
            "Organización y presentación de los diagramas, anexar descripción y estandarizar el formato"
        ],
    ]}
/>
