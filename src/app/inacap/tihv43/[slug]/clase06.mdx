# Diagrama Ishikawa

- También conocido como **Diagrama de Causa-Efecto** o **Diagrama de Espina de Pescado**

Es una herramienta visual de análisis que ayuda a identificar, explorar y mostrar las posibles causas de un problema específico. Fue creado por Kaoru Ishikawa en 1943.

### 3. ¿Cómo crear un Diagrama de Ishikawa?

    tableLayout="auto"

            "**Paso 2**",
            "Dibuja las categorías principales (6Ms) como ramas principales"

        ],
        [
            "**Paso 4**",
            "Para cada causa identificada, pregunta \"¿por qué ocurre esto?\" para encontrar sub-causas"
        ],
        [
            "**Paso 5**",
            "Analiza el diagrama completo para identificar las causas raíz más probables"
        ],
        
    ]}
/>

### Ejemplo práctico

**Problema:** Entrega tardías de software

- **Mano de obra:** Falta de programadores, equipo sin experiencia en la tecnología
- **Métodos:** Proceso de testing inadecuado, mala estimación de tiempos
- **Máquinas:** Servidores lentos, equipos obsoletos
- **Materiales:** Documentación incompleta, requisitos cambiantes
- **Medición:** No se monitorea el proceso adecuadamente
- **Medio ambiente:** Interrupciones constantes, espacio de trabajo ruidoso

## 4. Ventajas del Diagrama de Ishikawa

- Fácil de entender y crear
- Promueve el trabajo en equipo y la lluvia de ideas
- Identifica causas raíz, no solo síntomas
- Organiza visualmente información compleja
- Puede aplicarse a cualquier tipo de problema
