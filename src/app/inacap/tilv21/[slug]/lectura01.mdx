# Conceptos básicos en el Desarrollo de Software

- **Verificación:** Es el proceso de evaluar la calidad del software
- **Validación:** Consiste en comprobar que el software cumple con los requisitos del cliente a través de su funcionalidad. Se lleva a cabo una vez completada las verificaciones, al final del ciclo de desarrollo


## ¿Qué es la Verificación?
Es un proceso sistemático de comprobación del software durante su desarrollo. Este proceso incluye reuniones, inspecciones, revisiones y recorridos, para evaluar planos, códigos, documentos, especificaciones y requisitos, entre otros aspectos

**Pasos**

1. Verificación de los requisitos
2. Verificación del diseño
3. Verificación del código

## ¿Qué es la Validación?
Es un proceso utilizado para evaluar el software de acuerdo con las demandas del negocio o del cliente, tanto durante como al final del proceso de desarrollo. Este proceso revisa la aplicación final para comprobar si cumple las expectativas y los requisitos de los clientes.

- **Pruebas de caja blanca:** Ayudan a validar la aplicación mediante una serie de entradas de datos predefinidas. Los probadores comparan la salida de la aplicación con los valores de entrada para verificar si el software produce los resultados esperados
- **Pruebas de caja negra:** Se consideran 3 variables importantes, los valores de entrada, los valores de salida esperados y los valores de salida reales

## Errores, defectos y fallas

- **Errores:** Son acciones humanas que conducen a problemas potenciales de software
- **Defectos:** Son problemas reales en el código o diseño del software que pueden detectarse durante las pruebas
- **Fallas:** son problemas observados por los usuarios finales cuando utilizan el software en un entorno de producción
