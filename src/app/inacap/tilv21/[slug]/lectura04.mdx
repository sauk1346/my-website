# Síntesis de la Unidad 01

1. Diferenciación entre: error, defecto y fallo
2. Diferenciación entre: verificación y validación


## Definiciones

1. **Error:** problema cometido por el desarrollador
2. **Defecto:** anomalía en el software que puede llevar a un fallo de ejecución
3. **Fallo:** consecuencia de un defecto
4. **Verificación:** asegura que el software se construya correctamente
5. **Validación:** asegura que se cumpla con lo que el cliente necesita

## Estándares
Proporcionan marcos y directrices para la ejecución de pruebas, ayudando a asegurar que el software cumpla con los requisitos funcionales, no funcionales y de seguridad

1. **ISO/IEC 29119**
2. **IEEE 829**
3. **ISTQB**
4. **ISO 25010**
5. **OWASP**

|Descripción|Estándares|
|---|---|
|Aseguran calidad|ISO/IEC 29119, IEEE 829|
|Ofrece un marco estructurado para especificar y evaluar la calidad del software|ISO/IEC 25000|
|Ayuda a gestionar eficientemente el ciclo de vida del ISO/IEC 25000|ISO/IEC12207|
|Ayudan a gestionar y ejecutar pruebas de manera eficiente, garantizando que el software sea seguri, fiabla y cumpla con los requisitos establecidos|ISTQB, OWASP|
