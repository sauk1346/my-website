# Alcances y definiciones de los tipos de testing

## Pruebas funcionales
Verifican que el software cumpla con las funciones especificadas en los requisitos del sistema

- Pueden ser **estáticas** o **dinámicas**
- En las **pruebas de caja negra**, se evalúa el software desde una perspectiva externa, probando entradas y salidas sin conocer la estructura interna del sistema
- En las **pruebas de caja blanca**, se analiza la lógica interna del software mediante inspección del código fuente y la ejecución de rutas de códito específicas

## Pruebas de integración
Verifican la interoperabilidad entre diferentes módulos o subsistemas del software, para garantizar que funcionen juntos de manera cohesiva como un sistema completo

## Pruebas de aceptación
Son realizadas por los usuarios finales o stakeholders para validar que el software cumpla con sus necesidades y expectativas.

Aseguran que el software sea intuitivo, fácil de usar y cumpla con los requisitos funcionales y no funcionales acordados.

## Pruebas exploratorias
Se llevan a cabo sin un plan de prueba formal. Permite a los testers explorar el software y descubrir errores de manera espontánea.

Son valiosas para identificar problemas que no pueden ser evidentes en pruebas estructuradas y para mejorar la calidad general del software
