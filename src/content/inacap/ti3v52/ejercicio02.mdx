6. En el switch:
    - VLAN = 172.20.X.2/24
    - Máscara /24 = 255.255.255.0
    - Default Gateway = 172.20.X.1

<Solution title="Solución Pregunta 6">
Configuración para `Switch0` (en el primer piso)

- Ir a: `Switch0` >> `CLI`
- Escribir las siguientes instrucciones:
<TerminalOutput withBg>
{`# entrar al modo privilegiado
enable

# entrar al modo de configuración global
configure terminal

# entrar a la interfaz virtual de administración
interface vlan 1

# asignar IP y máscara a la interfaz VLAN 1
ip address 172.20.30.2 255.255.255.0

# activar la interfaz
no shutdown

# salir de la interfaz
exit

# definir la puerta de enlace para comunicarse fuera de la subred
ip default-gateway 172.20.30.1`}
</TerminalOutput>

7. En el router:
    - GO/0 IP 172.20.x.1/24
    - Asigne su propio nombre (en Mayúsculas) como nombre al router. Ejemplo: JUAN
    - Asigne su propio apellido.cl como nombre de dominio para SSH. Ejemplo: perez.cl
    - Crear usuario con nombre: **admin** contraseña cifrada: **cisco-admin**
    - Contraseña cifrada para modo privilegiado: **cisco-ena**
    - Configurar clave RSA con largo de la clave 2048 bits
    - Habilite versión 2 de SSH
    - Tiempo máximo de inactividad 60 seg
    - Intentos de autenticación: 2
    - Habilite ssh en líneas vty
    - Al conectarse por SSH debe solicitar usuario y contraseña
    - Desde cualquier PC del primer piso, en Command Prompt, ejecute el comando `ssh -l admin 172.20.X.1`
    - Ingrese la contraseña de admin, debe aparecer el nombre del router

<Solution title="Solución Pregunta 7">
Configuración para `Router0` (en el segundo piso)

- Ir a: `Router0` >> `CLI`
- Escribir las siguientes instrucciones:
<TerminalOutput withBg>
{`# entrar al modo privilegiado
enable

# 1. configuración interfaz e IP
# entrar al modo de configuración global
configure terminal
interface GigabitEthernet0/0
ip address 172.20.30.1 255.255.255.0
# activar la interfaz
no shutdown
# salir de la interfaz
exit

# 2. asignar nombre al router
hostname ANTONIO

# 3. asignar nombre de dominio para SSH
ip domain-name morales.cl

# 4. crear usuario con contraseña cifrada
username admin secret cisco-admin

# 5. contraseña cifrada para modo privilegiado
enable secret cisco-ena

# 6. generar claves RSA de 2048 bits para SSH (ingresar 2048 cuando se solicite)
crypto key generate rsa 

# 7. habilitar versión 2 de SSH
ip ssh version 2

# 8. tiempo máximo de inactividad: 60 segundos
ip ssh time-out 60

# 9. intentos máximos de autenticación
ip ssh authentication-retries 2

# 10. habilitar SSH en líneas vty y solicitar usuario/contraseña
line vty 0 4
# deshabilitar todos los protocolos de acceso remoto
no transport input
# permitir solo SSH
transport input ssh
login local
# salir de las líneas vty
exit`}
</TerminalOutput>
