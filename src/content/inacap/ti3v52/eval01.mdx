2. En **primer piso** identifique, selecciones y conecte cada dispositivo indicado a continuación:

8. Configuraciones básicas y de seguridad SSH en el router del tercer piso: 
    - G0/0 IP = 192.168.X.1 con máscara /24 = 255.255.255.0
    - Asigne su propio nombre (en Mayúscula) como nombre al router. Ejemplo: JUAN
    - Asigne su propio apellido.cl como nombre de dominio para SSH. Ejemplo: perez.cl
    - Crear usuario con nombre: **admin2** contraseña cifrada: **cisco-flex**
    - Contraseña cifrada para modo privilegiado: **cisco-c2**
    - Configurar clave RSA con largo de la clave 1024 bits
    - Habilite versión 2 de SSH
    - Tiempo máximo de inactividad 60 seg
    - Intentos de autenticación: 2
    - Habilite ssh en líneas vty
    - Al conectarse por SSH debe solicitar usuario y contraseña
    - Desde cualquier PC del primer piso, en Command Prompt, ejecute comando: `ssh -l admin2 192.168.X.1`
    - Ingrese la contraseña de admin2, debe aparecer el nombre del router

<Solution title="Solución Pregunta 8">
Configuración para `Router0` (tercer piso)

- Ir a: `Router0` >> `CLI`
<TerminalOutput withBg>
{`# entrar al modo privilegiado
enable
configure terminal

# G0/0 IP = 192.168.18.1 con máscara /24 = 255.255.255.0
interface g0/0
ip address 192.168.18.1 255.255.255.0
no shutdown
exit

# asigne su propio nombre (en Mayúscula) como nombre al router
hostname ANTONIO

# asigne su propio apellido.cl como nombre de dominio para SSH
ip domain-name morales.cl

# crear usuario con nombre: admin2, contraseña cifrada: cisco-flex
username admin2 secret cisco-flex

# contraseña cifrada para modo privilegiado: cisco-c2
enable secret cisco-c2

# configurar clave RSA con largo de la clave 1024 bits (ingresar 1024 cuando se solicite)
crypto key generate rsa

# habilite versión 2 de SSH
ip ssh version 2

# tiempo máximo de inactividad 60 seg
ip ssh time-out 60

# intentos de autenticación: 2
ip ssh authentication-retries 2

# habilite SSH en líneas vty / al conectarse por SSH debe solicitar usuario y contraseña
line vty 0 4
no transport input
transport input ssh
login local
exit`}
</TerminalOutput>
