<TerminalOutput 
	withBg
	id="salida01"
	caption="Configuración router RENCA en CLI"
>
{`# entrar al modo privilegiado
enable

# entrar al modo de configuración global
configure terminal

# 1. asignar nombre al router
hostname RENCA

# 2. configuración interfaz G0/0 (LAN 1 - 16.000 IPs)
interface g0/0
# asignar IP (primera IP de LAN 1, /18)
ip address 130.130.0.1 255.255.192.0
# activar la interfaz
no shutdown
# salir de la interfaz
exit

# 3. configuración interfaz G0/1 (LAN 2 - 4.000 IPs)
interface g0/1
# asignar IP (primera IP de LAN 2, /20)
ip address 130.130.64.1 255.255.240.0
# activar la interfaz
no shutdown
# salir de la interfaz
exit

# 4. configuración interfaz S0/0/0 (WAN 1 - 2 IPs)
interface s0/0/0
# asignar IP (primera IP de WAN 1, /30)
ip address 130.130.84.1 255.255.255.252
# activar la interfaz
no shutdown
# salir de la interfaz
exit

# volver al modo privilegiado
end

# guardar configuración en NVRAM
wr`}
</TerminalOutput>

<P indent={2}>
**Paso 08: Rutas estáticas**
</P>

<P indent={3}>
Para el enrutamiento estático se utiliza la siguiente instrucción:
</P>

<TerminalOutput withBg>
{`# instrucción para enrutamiento estático
ip route <red destino> <mascara destino> <IP next-hop>`}
</TerminalOutput>

<List level={1} indent={2}>
- **red destino:** la red a la que quieres llegar 
- **máscara destino:** la máscara de esa red
- **IP next-hop:** la IP de la interfaz del router vecino por donde sale el tráfico
</List>
