## Parte 1: Configurar ID del router

1. Inicie el proceso de enrutamiento OSPF en los 3 routers. Usar ID de proceso **10**

```bash
Router(config)# router ospf process-id
```

2. Utilice el comando `router-id` para establecer los identificadores OSFP de los 3 routers de la siguiente manera
    - R1: 1.1.1.1
    - R2: 2.2.2.2
    - R3: 3.3.3.3

    Use el comando
```bash
Router(config-router)# router-id rid
```

## Parte 2: Configurar redes para enrutamiento OSPF

### Paso 1: Configurar redes para enrutamiento OSPF mediante comandos de red y máscaras comodín
- ¿Cuántas instrucciones se requieren para configurar OSFP para enrutar todas las redes conectadas al router R1?
- La LAN conectada al router R1 tiene una máscara /24. ¿Cuál es el equivalente de esta máscara en la representación decimal de puntos?
- Reste la máscara de subred decimal punteada de 255.255.255.255. ¿Cuál es el resultado?
- ¿Cuál es el equivalente decimal punteado de la máscara de subred /30?
- Resta la represetntación decimal con puntos de la máscara /30 de 255.255.255.255. ¿Cuál es el resultado?

1. Configure el proceso de enrutamiento de R1 con las instrucciones de red y las máscaras comodín necesarias para activar el enrutamiento OSPF para todas las redes conectadas. Los valores de instrucción de red deben ser las direcciones de red o subred de las redes configuradas.

```bash
Router (config-router)# network <máscara comodín> area <area id>
```

2. Compruebe que OSPF se ha configurado correctamente mostrando la configuración en ejecución. Si encuentra un error, elimine la sentencia network utilizando el comando `no` y vuelva a configurarla.

<Solution title="Solución Parte 2 (Paso 1)">

- **¿Cuántas instrucciones se requieren para configurar OSFP para enrutar todas las redes conectadas al router R1?**
<P indent={3}>
**Resp:** 3 instrucciones, una por cada red conectada: la LAN y las 2 seriales
</P>
- **La LAN conectada al router R1 tiene una máscara /24. ¿Cuál es el equivalente de esta máscara en la representación decimal de puntos?**
<P indent={3}>
**Resp:** 255.255.255.0
</P>
- **Reste la máscara de subred decimal punteada de 255.255.255.255. ¿Cuál es el resultado?**
<P indent={3}>
**Resp:** 0.0.0.255
</P>
- **¿Cuál es el equivalente decimal punteado de la máscara de subred /30?**
<P indent={3}>
**Resp:** 255.255.255.252
</P>
- **Resta la represetntación decimal con puntos de la máscara /30 de 255.255.255.255. ¿Cuál es el resultado?**
<P indent={3}>
**Resp:** 0.0.0.3
</P>
---

## Parte 3: Configure las interfaces Pasivas

OSPF enviará su tráfico de protocolo fuera de todas las interfaces que participan en el proceso OSPF. En los vinculos que no están para otras redes, como LAN, este tráfico innecesario consume recursos. El comando `passive-interface` evitará que el proceso OSPF envíe tráfico de protocolo de enrutamiento inncesario fuera de interfaces LAN.

¿Qué interfaces en R1, R2 y R3 son interfaces LAN?

Configure el proceso OSPF en cada u no de los 3 routers con el comando `passive-interface`

```bash
Router(config-router)# passive-interface interface
```

<Solution title="Solución Parte 3">

- **¿Qué interfaces en R1, R2 y R3 son interfaces LAN?**
<P indent={3}>
**Resp:** R1->g0/0/0, R2->g0/0/0, R3->g0/0/0
</P>
